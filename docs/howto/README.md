# Howtos

Hands-on guides for projects looking to **adopt Matrix as their primary
community channel**. Each guide is opinionated, short, and starts from
"I have nothing set up yet" — not from "let me explain federation."

These guides are stubs. If you have run any of these flows in production
and have advice, fixes, or screenshots, open a PR. The goal is "you can
follow this end-to-end in an afternoon," not "comprehensive reference."

## What "stub" means here

Each file below has:

- A **scope** section — what you'll have at the end.
- A **prereqs** section — what you need going in.
- A numbered **steps** list with the right shape but TODO markers where
  the prose still needs to be written.
- A **gotchas** section that notes the things that are *known* to bite
  even if the steps aren't fleshed out yet.
- A **further reading** section pointing at the canonical Matrix docs
  so a stub is still useful.

Filling these in is mostly a writing exercise — the structure is not
the bottleneck.
